Карточка номера

Общие сведения

Форма "Карточка номера" предоставляет пользовательский интерфейс к данным одной записи реестра номеров.

Область идентификации

Для пользователей, имеющих компетенции "Руководитель отдела ППИ", "Сотрудник отдела ППИ" для редактирования доступны поля и управляющие элементы, позволяющие однозначно идентифицировать номер (выпуск) газеты:

  • серия нумерации (группа кнопок)
  • номер выпуска (поле ввода)
  • дата выхода номера (поле ввода)
  • точность даты выхода (группа кнопок)
  • ложный номер (чекбокс)
  • ненумерованный выпуск (чекбокс)
  • примечание (поле ввода).
Возможность редактирования сведений о валовой нумерации и тираже пользователям с указанными компетенциями не предоставляется.

Редактирование серии нумерации

Осуществляется простым выбором (клик лкм) по "кнопке" номера серии. Выбрать можно только один номер. Выбранный номер подсвечивается:

Редактирование номера выпуска

Поле "Номер выпуска" и чекбокс "Ненумерованный выпуск" связаны между собой. При взводе флага в поле "Номер" автоматически вносится :" Ненумер. вып." (лидирующие пробелы важны!). При вводе любого значения флаг "Ненумерованный выпуск" сбрасывается. В записи номер выпуска будет сохранен с необходимым количеством лидирующих пробелов.

Флаг "Не получен"

Все записи, создаваемые сотрудниками отдела ППИ, автоматически помечаются флагом "Не получен", возможность сброса флага пользователям с компетенциями "Руководитель отдела ППИ", "Сотрудник отдела ППИ" не предоставляется.

Флаг "Ложный номер"

Взвод этого флага позволяет сохранить в базе данных несколько выпусков с одинаковыми номерами за год выпуска.

Флаг "Прекращение издания"

Возможность изменения состояния этого флага пользователям с компетенциями "Руководитель отдела ППИ", "Сотрудник отдела ППИ" не предоставляется.

Редактирование даты выпуска

Дата выпуска может быть указана с точностью до дня, месяца или года.

Точность даты указывается простым выбором (клик лкм) по "кнопке" "День", "Месяц" или "Год". Выбранный вариант подсвечивается.

При выборе точности "День" доступен прямой ввод даты (с клавиатуры). При нажатии на кн.обзора будет доступен выбор из календаря:

При выборе точности "Месяц" или "Год" доступен ввод только с использованием кнопки обзора - через диалоги выбора месяци или года:

Допускается вводить дату выпуска за пределами года выпуска, но для этого потребуется дополнительное подтверждение.

Область учета количества принятых экземпляров

Область содержит:

  • Сведения о цене (поле ввода)
    Возможность редактирования сведений об учетной цене пользователям с компетенциями "Руководитель отдела ППИ", "Сотрудник отдела ППИ" не предоставляется.
  • Сведения о количестве поступивших экземпляров.
    Эта область реализована в форме таблицы, позволяющей сохранять информацию о поступающих экземплярах в разрезе партий (учетных документов).
  • Итоговые сведения о количестве поступивших экземпляров.

Порядок ввода количества принятых экземпляров

Для добавления информации о количестве поступивших номеров:
  • Добавить новую позицию в таблицу "Количество экземпляров, принятое на хранение".

    Строки добавляются кликом по символу +:

  • Заполнить данные в строке.

    Если учетный документ (накладная) уже внесен в базу данных, его можно выбрать, "провалившись" в клетку в столбце "Учетный документ" (клавиша Enter или клик лкм по кнопке обзора в клетке).
    Если учетный документ ещё не внесен - оставьте это поле незаполненным.

    В столбец "Кол-во экз." данные вводятся с клавиатуры.

    Конпка обзора в клетке столбца "Кол-во экз." позволяет открыть форму-редактор записи "Единица хранения".

Область служебной информации

Данные в этой области не предназначены для редактирования. Область содержит сведения:

  • Метку факта, что запись создана сотрудником группы ЛГС.
  • ID записи.
    Это внутренний идентификатор записи. Он вычисляется системой и недоступен для редактирования.
  • Сведения о регистрации номера.
  • Сведения о пользователе, создавшем запись, дате и времени создания.
    Сведения проставляются системой и недоступны для редактирования.
  • Сведения о пользователе, последним изменившим запись, дате и времени изменения.
    Сведения проставляются системой и недоступны для редактирования.

Прочие управляющие элементы

  • элемент окна формы - закрывает форму и сохраняет запись после дополнительного подтверждения. Если изменений данных не было - просто закрывает окно.
  • "Закрыть" (кнопка) - закрывает форму и сохраняет запись без дополнительного подтверждения.

    В любом случае перед сохранением выполняется проверка данных на корректность. Без исправления некорректных данных форма не закроется.

  • "Отмена" (кнопка) - закрывает форму без сохранения изменений.
