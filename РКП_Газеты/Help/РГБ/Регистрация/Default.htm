АС Учет периодических печатных изданий

Обзор функционального модуля "Регистрация газет и МП"

Общие сведения

Назначение функционального модуля - обеспечение ввода, редактирования, сохранения в базе данных, аналитической обработки информации о газетах и МП, выпускающихся на территории РФ.

Главное окно программы

Функциональный модуль "Учет газет и МП" имеет две преднастроенные интерфейсные схемы, различающиеся составом доступных команд (опций). Применяемый вариант интерфейсной схемы связан с правами пользователя.

Пользовательские сценарии

Работа с приложением предполагает несколько пользовательских сценариев.

  • Создание / редактирование карточек изданий

    Сценарий предполагает создание и редактирование записей, содержащих основную информацию об издании. Действия по созданию, поиску ранее созданных записей осуществляются в картотеке - реестре изданий. Ввод и редактирование информации выполняется в форме Карточка издания.

    Кнопка вызова реестра изданий расположена в начале панели инструментов и в разделе «Данные» главного меню. Если в настройку прав пользователя не были внесены изменения, окно "Реестр изданий" будет открываться каждый раз при старте сессии.

    Общие принципы работы с картотекой как с пользовательским интерфейсом обращения к записям базы данных описано в статье Окно картотеки.

    Особенности работы с картотекой изданий изданий - в статье Реестр газет и мелкой печати.

  • Ввод информации о номерах (выпусках) изданий

    Сценарий предполагает ввод и редактирование информации о поступающих на хранение номерах (выпусках) изданий в разрезе ранее созданной записи об издании.

    Действия выполняются в форме Номера газеты. Форма доступна исключительно через карточку издания.

  • Анализ выполненного объема ввода / редактирования данных

    Сценарий предполагает формирование отчетов о количестве введенных и измененных записей базы данных.

    Предусмотрено три вида отчетов:

    • Отчет о работе библиографа.
      Отчет доступен любому сотруднику ОРГП. Содержит информацию только о работе пользователя, под чьим именем была открыта сессия работы. Результаты построенного после закрытия формы не сохраняются в базе данных.
    • Отчеты о работе отдела РГП.
      Отчеты доступны только сотрудникам с првами руководителя отдела ОРГП, директора департамента и администратора прикладной системы. Построенные отчеты сохраняются в базе данных.
    • Отчеты о работе отдела ППИ.
      Отчеты доступны только сотрудникам с првами руководителя отдела ОРГП, руководителя отдела ОППИ, директора департамента и администратора прикладной системы. Построенные отчеты сохраняются в базе данных.

  • Формирование аналитических выборок

    Сценарий предполагает формирование отчетов, извлекающих информацию о зарегистрированных изданиях и номерах (выпусках) в различных разрезах по произвольно задаваемым правилам отбора данных.

    Предусмотрены отчеты:

    • Справка по изданиям;
    • Справка о статистических показателях;
    • Справки по последнему номеру;
    • Отчеты о пополнении базы данных.

  • Формирование статистических отчетов

    Сценарий предполагает формирование типовых отчетов, включаемых в ежегодные обзоры:

    • Выпуск газет по месту издания;
    • Выпуск газет по целевому назначению и читательскому адресу;
    • Выпуск газет по тематике;
    • Выпуск газет на языках народов мира;
    • Выпуск газет в Российской Федерации;
    • Выпуск газет по периодичности;
    • Выпуск газет субъектами РФ по целевому назначению и читательскому адресу;
    • Выпуск всероссийских (общенациональных) газет в Российской Федерации;
    • Выпуск газет по территории распространения;

  • Подготовка данных для "Летописи периодических и продолжающихся изданий"

    Действия по подготовке данных выполняются в картотеке "Выпуски ЛППИ".

  • Экспорт данных в формат RusMarс

    Действия по подготовке данных выполняются в картотеках "Сеансы экспорта газет в БД РКП" и "Сеансы экспорта номеров в БД РКП".
